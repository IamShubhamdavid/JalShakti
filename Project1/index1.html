Day 3: File Path System in HTML | Boilerplate | DIV + Class + ID |

The day 3 was all about File Path in HTML , HTML BoilerPlate Code , DIV + Class + ID .

Why Relative Paths Matter?

  • Relative paths work across different environments (local vs server).
  • Absolute local paths like URL starting with http:// or https:// break when deployed.

    • Multi-Page Websites

  • Use navigation system with tags
  • Break content into multiple pages and maintain consistent across pages

    • Why Boilerplate Matters?

  • Ensures standards-compliant rendering
  • Prevents character encoding issues
  • Viewport meta tag: Enables responsive mobile design

    • Div | Class | Id

  • Div: Generic block-level container
  • Id: Unique identifier(use once per page)
  • Class: Reusable classifier (use multiple times)
  • Header: Top section (logo, navigation)
  • Footer: Bottom section (copyright, links)
  • main: Primary content area (unique per page)
    •      		Key Concepts:
    • Relative Paths uses for local resources and Absolute Paths use for external resources
    • Always include the HTML boilerplate for consistent rendering
    • Create consistent navigation across multi-page sites
    • Use classes and IDs to target elements for styling
